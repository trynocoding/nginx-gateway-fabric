# NGF examples 二八源码学习路线

> [!summary] 选取结论
> 本语料从 `examples/` 的 130 个文件中选取 27 个高信息密度文件，覆盖 18/22 个场景族（约 82%）。它不是“文件数量的 80%”，而是用约 21% 的文件覆盖 Gateway/Route、L4/L7、TLS、跨命名空间、过滤器、认证、策略、WAF 与 Helm 安装等核心实现机制。

## 阅读主线

1. 先读 cafe 的 Gateway、Route、工作负载三件套，理解直接输入与间接依赖。
2. 再读 HTTP 高级匹配、流量切分、gRPC、TCP/UDP，建立协议分层。
3. 接着读 TLS、ReferenceGrant、BackendTLSPolicy 和 ExternalName，理解信任与解析边界。
4. 最后读 filter/policy/WAF/Helm，理解扩展点和能力开关。

## 有意省略的重复项

- 各目录 README：它们是操作说明，不是控制器输入；需要时回看原文即可。
- 重复的 `app.yaml`/`cafe.yaml`：Deployment/Service 的边界已由 cafe 基线说明。
- 仅名称、路径或策略值不同的同构 YAML：由对应代表笔记中的字段谱系和失败路径覆盖。
- Helm default/nodeport/inference 等组合变体：保留 experimental-plus 与 snippets 两个高差异样本，其他可按同一模板谱系迁移。
- UDP、请求头、Basic Auth 与 ClientSettings：分别由 TCP stream、响应头 modifier、外部认证和 ProxySettings 的相邻机制覆盖；需要时可沿索引中的公共主链迁移分析方法。
