# 29 泛型函数与类型推断

> [!abstract]
> 泛型函数把“算法不变、参与类型可变”写进类型系统。调用时编译器通常从普通实参反推类型实参；推不出来或需要主动指定目标类型时，才写 `F[T](...)`。

## 学习目标与前置

读完后应能：

- 读懂 `func F[T Constraint](v T)`、类型形参、类型实参和约束；
- 判断类型推断来自哪里、何时失败、何时应显式写类型实参；
- 写出返回零值的通用算法，而不借助 `any` 和类型断言；
- 从 NGF 的 `helpers.GetPointer` 追到真实调用者与测试。

前置：[[08-指针nil与可选字段|指针、nil 与可选字段]]、[[24-接口的隐式实现]]。本章只讲泛型函数；约束细节见 [[30-any与comparable]]、[[31-方法约束]]、[[32-类型集合波浪号与联合约束]]。

## 1. 从函数到泛型函数

普通函数的参数类型在声明时已经固定：

```go
func IntPointer(v int) *int { return &v }
func StringPointer(v string) *string { return &v }
```

两个函数的控制流完全相同。泛型把变化的类型提为类型形参：

```go
func Pointer[T any](v T) *T { return &v }
```

- `[T any]` 是类型形参列表；`T` 只在这次实例化中代表某个具体类型；
- `any` 是约束，表示这里不需要对 `T` 做额外操作；
- `(v T)` 和 `*T` 像普通类型一样使用 `T`；
- `Pointer(42)` 会实例化出可理解为 `func(int) *int` 的版本。

> [!note]
> 泛型不是“运行时装进一个万能盒子”。类型检查发生在编译期，`Pointer(42)` 的结果静态类型就是 `*int`。

## 2. 类型推断的来源

最常见的是从函数实参推断：

```go
p1 := Pointer("ngf")      // T = string
p2 := Pointer[int64](42)  // 显式指定 T，42 转为 int64
```

多个形参共享一个 `T` 时，实参必须能得到一致的实例化：

```go
func Pair[T any](a, b T) [2]T { return [2]T{a, b} }

_ = Pair(1, 2)          // T = int
_ = Pair[int64](1, 2)   // T = int64
// _ = Pair(1, "two")   // 编译失败：无法让同一个 T 同时满足 int 和 string
```

返回值本身通常不能替一个“完全没出现在参数中”的类型形参完成推断：

```go
func Zero[T any]() T { var z T; return z }

// z := Zero()     // 编译失败：cannot infer T
z := Zero[string]()
_ = z
```

## 3. 可独立运行的最小 demo

**可运行 demo（Go 1.26.0，保存为 `main.go`）：**

```go
package main

import "fmt"

func Pointer[T any](v T) *T { return &v }

func Map[S, D any](in []S, f func(S) D) []D {
	out := make([]D, len(in))
	for i, v := range in {
		out[i] = f(v)
	}
	return out
}

func Zero[T any]() T {
	var zero T
	return zero
}

func main() {
	name := Pointer("gateway") // 推断 T=string
	ports := Map([]int{80, 443}, func(v int) string {
		return fmt.Sprintf(":%d", v)
	}) // 推断 S=int、D=string

	fmt.Println(*name, ports, Zero[bool]())
}
```

运行：

```bash
gofmt -w main.go
go run main.go
# gateway [:80 :443] false
```

## 4. 四种常用模式

### 4.1 值转指针

API 中 `*string`、`*bool` 常表示“未设置”和“显式零值”的区别。`Pointer[T]` 避免为每个命名类型写临时变量。

### 4.2 集合变换

`Map[S, D]` 保持输入、输出元素的静态类型。若不同类型的转换规则由调用者提供，泛型比 `[]any` 更安全。

### 4.3 查找并返回 `T` 的零值

当签名是 `(T, bool)` 或 `(T, error)`，失败路径可用：

```go
var zero T
return zero, false
```

不能写 `return nil, false`，因为 `T` 也可能是 `int`。

### 4.4 显式类型实参承担转换

`Pointer[gatewayv1.PortNumber](8080)` 同时说明目标命名类型并完成可表示常量的转换。若推断结果已经准确，重复写 `[string]` 只会增加噪音。

## 5. 编译边界与误区

| 情况 | 结果 | 修正 |
|---|---|---|
| 类型形参只出现在结果中 | 无法推断 | 写 `Zero[string]()` |
| 一个 `T` 收到互不兼容的实参 | 无统一实例化 | 拆成 `A, B` 或修正 API |
| `T any` 内写 `a == b` | 编译失败 | 使用 `T comparable` |
| 想对 `T` 调 `Name()` | 编译失败 | 约束中声明 `Name() string` |
| 为动态插件选择实现 | 泛型解决不了运行时选择 | 使用接口/注册表 |

泛型函数可以取值和返回值，但不能凭空获得约束未声明的操作。约束不是注释，而是函数体的权限清单。

## 6. NGF：`GetPointer` 如何进入生产路径

**NGF 缩写源码（不是独立 demo）：**

```go
// internal/framework/helpers/helpers.go
func GetPointer[T any](v T) *T {
	return &v
}
```

一个代表性生产调用在 `internal/controller/handler.go:getGatewayAddressesForStatus`：

```go
Type: helpers.GetPointer(gatewayv1.IPAddressType),
```

数据关系是：Kubernetes `Service` 地址 → 构造 `GatewayStatusAddress` → 编译器从 `gatewayv1.IPAddressType` 推断 `T` → 得到该命名类型的指针 → 写入 Gateway 状态的可选 `Type` 字段。这里选择泛型，是因为算法只做“取值地址”，不依赖接口行为，也不需要反射。

`internal/controller/state/dataplane/convert_test.go:TestConvertHTTPRequestRedirectFilter` 还展示了两种调用：能推断时写 `GetPointer("https")`；需要目标 API 命名类型时写 `GetPointer[v1.PortNumber](8443)`。`internal/framework/helpers/helpers_test.go:TestEqualPointers` 通过大量 `GetPointer` 构造用例，间接覆盖它与可选字段比较的组合。

> [!warning]
> `GetPointer(v)` 返回的是形参副本的地址，不是原变量的地址。之后修改原变量不会修改 `*p`。这正适合构造 API 字段，但不适合期望别名共享的场景。

## 7. 为什么不用接口、反射或 `any`

- 泛型：同一个取地址算法适用于所有静态类型，且结果保留 `*T`；本例首选。
- 接口：适合调用共同方法；这里没有共同“行为”。
- `any`：若返回 `*any`，调用者还要断言，命名类型信息也丢失。
- 反射：只有运行时才知道类型元数据时才值得付出复杂度；这里编译器已经知道 `T`。

迁移边界：若将来函数要根据类型注册动态处理器，应迁移到接口或注册表；不要在泛型函数里用反射模拟动态分派。

## 8. 练习与检查点

1. 写 `First[T any]([]T) (T, bool)`，空切片返回零值。
2. 为什么 `Pointer(nil)` 无法推断？如何得到 `*error` 指向一个 nil 接口值？
3. 将 demo 的 `Map` 调用改成把 `[]string{"80"}` 转成 `[]int`，失败时返回错误。签名应如何变化？

答案/检查点：

```go
func First[T any](xs []T) (T, bool) {
	if len(xs) > 0 {
		return xs[0], true
	}
	var zero T
	return zero, false
}
```

第 2 题：无类型的 `nil` 不提供 `T`；可写 `Pointer[error](nil)`。第 3 题应让变换函数返回 `(D, error)`，`Map` 自身返回 `([]D, error)`，并在失败处包装下标语境；不要用 panic 表示用户输入解析失败。
