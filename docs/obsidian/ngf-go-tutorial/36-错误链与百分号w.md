# 36 %w 与错误链

> [!abstract] 本章唯一知识点
> fmt.Errorf 使用 %w 在增加语境的同时保留可遍历的原因链。

## 前置与完成标准

前置：[[35-泛型接口反射选择边界]]。学完应能解释“%w 与错误链”，并在 NGF 中定位 `Write`，说清调用效果与适用边界。本章只做源码导读，片段不承诺可独立运行。

## 最小模型

阅读时先判断类型、所有权和生命周期由谁规定，再读语法；这样能把记忆中的写法重新连回工程约束。

## NGF 生产代码证据

**internal/framework/file/file.go · Write（原样摘录）**

```go
return fmt.Errorf("failed to create file %q: %w", file.Path, err)
```

- 定义：`ngf:internal/framework/file/file.go:Write`
- 精简链路：底层文件错误 → 加入路径与动作 → 上层返回或聚合 → 日志获得可读上下文。
- 测试佐证：`internal/framework/file/file_test.go`
- 证据范围：源码事实固定于 `df175d68`；测试文件用于确认行为边界，不把框架推断写成项目事实。

## 工程取舍与边界

每一层只补充本层动作，不重复记录日志。

> [!warning] 常见误解与迁移边界
> 跨抽象边界可直接复用；面向用户的敏感路径要脱敏。误解是 %v 与 %w 对 errors.Is/As 等价。

复用判定：明确可复用的做法可直接采用；带条件的做法必须先满足相同输入、所有权与失败语义；指出不要或不可的部分不应照搬。

## 心智模型与下一步

一句话心智模型：**%w 与错误链不是孤立语法，而是 `Write` 所在边界用来表达约束的工具。**
