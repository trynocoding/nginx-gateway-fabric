# 51 io.Reader/io.Writer 与资源所有权

## 语法

Reader/Writer 以流能力解耦数据源和目的地；接口不自动定义 Close，资源所有权必须另行约定。

**说明性片段：**

```go
func Copy(dst io.Writer, src io.Reader) error {
	_, err := io.Copy(dst, src)
	return err
}
```

## NGF 中的应用

位置：`ngf:internal/framework/file/os_filemanager.go:StdLibOSFileManager.Copy`

**原样源码：**

```go
func (s *StdLibOSFileManager) Copy(dst io.Writer, src io.Reader) error {
```

调用者打开源/目标并拥有 Close → Copy 只传输字节 → 返回写入错误。
