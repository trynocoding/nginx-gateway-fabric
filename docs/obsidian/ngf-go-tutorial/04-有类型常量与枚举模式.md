# 04 有类型常量与枚举模式

> [!abstract] 本章唯一知识点
> Go 没有 enum 关键字；常用“定义底层类型 + const 集合”获得类型检查和可读常量名。

## 前置与完成标准

前置：[[03-变量声明与零值]]。学完应能解释“有类型常量与枚举模式”，并在 NGF 中定位 `ControllerLogLevel`，说清调用效果与适用边界。本章只做源码导读，片段不承诺可独立运行。

## 最小模型

阅读时先判断类型、所有权和生命周期由谁规定，再读语法；这样能把记忆中的写法重新连回工程约束。

## NGF 生产代码证据

**apis/v1alpha1/nginxgateway_types.go · ControllerLogLevel（原样摘录）**

```go
type ControllerLogLevel string
```

- 定义：`ngf:apis/v1alpha1/nginxgateway_types.go:ControllerLogLevel`
- 精简链路：CRD JSON 字符串 → ControllerLogLevel → validateLogLevel switch → 日志级别更新。
- 测试佐证：`internal/controller/config_updater_test.go`
- 证据范围：源码事实固定于 `df175d68`；测试文件用于确认行为边界，不把框架推断写成项目事实。

## 工程取舍与边界

API 类型用有类型字符串常量，并由 Kubebuilder 枚举和运行期校验共同约束。

> [!warning] 常见误解与迁移边界
> 可复用到有限字符串域；新增值必须同步 schema、转换、校验和测试。误解是 const 集合会自动拒绝任意强制转换值。

复用判定：明确可复用的做法可直接采用；带条件的做法必须先满足相同输入、所有权与失败语义；指出不要或不可的部分不应照搬。
