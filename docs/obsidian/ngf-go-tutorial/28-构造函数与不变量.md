# 28 构造函数与不变量建立

> [!abstract] 本章唯一知识点
> Go 构造函数只是普通函数；价值在于初始化 map/channel/context 等不允许为零的状态。

## 前置与完成标准

前置：[[27-Functional-Options]]。学完应能解释“构造函数与不变量建立”，并在 NGF 中定位 `NewDeploymentBroadcaster`，说清调用效果与适用边界。本章只做源码导读，片段不承诺可独立运行。

## 最小模型

阅读时先判断类型、所有权和生命周期由谁规定，再读语法；这样能把记忆中的写法重新连回工程约束。

## NGF 生产代码证据

**internal/controller/nginx/agent/broadcast/broadcast.go · NewDeploymentBroadcaster（原样摘录）**

```go
listeners:         make(map[string]storedChannels),
```

- 定义：`ngf:internal/controller/nginx/agent/broadcast/broadcast.go:NewDeploymentBroadcaster`
- 精简链路：调用构造函数 → 初始化 channels/map/child context → 启动 subscriber 和 publisher → 返回可用对象。
- 测试佐证：`internal/controller/nginx/agent/broadcast/broadcast_test.go`
- 证据范围：源码事实固定于 `df175d68`；测试文件用于确认行为边界，不把框架推断写成项目事实。

## 工程取舍与边界

返回前建立所有并发不变量，调用者无需知道内部启动顺序。

> [!warning] 常见误解与迁移边界
> 有资源或不变量的类型应强制走构造器；纯数据 struct 可用字面量。误解是名字叫 New 就由语言提供特殊语义。

复用判定：明确可复用的做法可直接采用；带条件的做法必须先满足相同输入、所有权与失败语义；指出不要或不可的部分不应照搬。
