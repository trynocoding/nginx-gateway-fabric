# 10 数组与切片的类型差异

> [!abstract]
> 数组 `[N]T` 的长度属于类型并包含 N 个元素；切片 `[]T` 是描述一段底层数组窗口的值，含指针、长度和容量。数组赋值复制全部元素，切片赋值通常共享底层数组。

## 学习目标与前置

- 声明、索引和遍历数组/切片；
- 理解 `[32]byte` 与 `[]byte` 为什么不能直接赋值；
- 使用切片表达式 `a[:]` 从数组建立视图；
- 读懂 NGF `sha256.Sum256` → `sum[:]` → hex 的数据转换。

前置：[[06-struct与复合字面量]]。切片共享细节见 [[12-切片共享与防御性复制]]。

## 1. 类型与表示

```go
var a [3]int       // 类型是 [3]int，值为 [0 0 0]
b := [3]int{1, 2, 3}
s := []int{1, 2, 3} // 类型是 []int
```

`[3]int` 与 `[4]int` 是不同类型。数组长度编译期固定，数组可比较（若元素可比较）。slice 长度运行期变化，只能与 nil 比较，不能整体 `==`。

### 从数组切出 slice

```go
s := b[:]     // 覆盖整个数组
part := b[1:] // 从索引 1 到末尾
```

这通常不复制元素；slice 与数组共享存储。修改 `s[0]` 会修改 `b[0]`。

### 数组赋值与函数参数

```go
c := b
c[0] = 99 // b[0] 不变
```

数组是值；传 `[32]byte` 参数会复制整个数组。slice 参数复制的只是 slice header，元素仍共享。

## 2. 可独立运行的最小 demo（Go 1.26）

```go
// 可运行示例；Go 1.26.0
package main

import "fmt"

func main() {
	array := [3]int{10, 20, 30}
	arrayCopy := array
	sliceView := array[:]

	arrayCopy[0] = 100
	sliceView[1] = 200

	fmt.Println("array", array)
	fmt.Println("copy", arrayCopy)
	fmt.Printf("slice len=%d cap=%d value=%v\n", len(sliceView), cap(sliceView), sliceView)
}
```

```bash
gofmt -w main.go
go run main.go
# array [10 200 30]
# copy [100 20 30]
# slice len=3 cap=3 value=[10 200 30]
```

失败例：`var s []int = array` 无法编译，必须写切片表达式 `array[:]`，因为数组和 slice 是不同类型。

## 3. 常用模式

### 3.1 固定尺寸的协议/摘要用数组

哈希摘要、IP 字节数、矩阵维度需要在类型中保留长度时用 `[N]byte`，编译器能阻止错长数据混入。

### 3.2 变长集合和 API 参数用 slice

路由列表、header 列表等长度运行时决定，slice 便于 append、截取和传递。

### 3.3 数组到 slice 作为适配边界

标准库函数可能返回固定数组，而编码器接收 `[]byte`；用 `sum[:]` 建立零拷贝视图。

### 3.4 小数组作为 map key

元素可比较时数组可作为 map key，例如 `[16]byte` ID；slice 不可比较，需转 string/数组或自定义 hash。

## 4. NGF：`URLHash` 中的三次类型转换

焦点：`ngf:internal/framework/helpers/helpers.go:URLHash`。

```go
// NGF 原样短源码，不是独立 demo
func URLHash(rawURL string) string {
	sum := sha256.Sum256([]byte(rawURL))
	return hex.EncodeToString(sum[:])[:16]
}
```

逐步拆解：

1. `[]byte(rawURL)` 把不可变 string 转成字节 slice；
2. `sha256.Sum256` 接收 `[]byte`，返回固定 `[32]byte`；
3. `sum[:]` 创建覆盖摘要数组的 `[]byte` 视图；
4. `hex.EncodeToString` 接收 slice，生成 64 字符十六进制 string；
5. `[:16]` 取前 16 个 ASCII 字节，形成稳定、文件名安全的组件。

为什么 `Sum256` 返回数组：SHA-256 输出长度固定，类型直接表达 32 字节不变量。为什么编码器接收 slice：它可以编码任意长度字节序列。这是数组和 slice 各自在 API 中最自然的责任。

`internal/framework/helpers/helpers_test.go` 对固定输入验证 `URLHash` 的确定输出、长度或差异行为；生产调用将结果用于稳定 key/文件名组件。注意只取 16 个 hex 字符降低了标识长度，也降低抗碰撞位数；它不是密码学身份或权限令牌。

## 5. 边界与误区

- `len(array)` 是类型固定长度；`len(slice)` 是当前窗口长度；
- `cap(slice)` 从起点算到底层数组末端，不等于总数组长度；
- `array[:]` 共享数据，不是转换副本；需要隔离时 `slices.Clone` 或 copy；
- `[]byte(string)` 通常产生可修改字节副本，反向 `string(bytes)` 也应按转换看待；
- 不要用 `[1000000]T` 按值到处传递；大固定缓冲需评估指针、slice 和分配策略。

## 6. 迁移判断

- **直接复用**：固定摘要/协议字段用数组；变长集合用 slice。
- **条件复用**：数组切片后零拷贝传递，前提是被调用方不会保存或修改超出所有权约定的数据。
- **不可照搬**：截短哈希作为安全唯一标识；NGF 此处注释只承诺稳定、文件系统安全的组件。

## 7. 练习与答案

1. `[2]int` 能赋给 `[3]int` 吗？不能，长度属于类型。
2. `array[:]` 修改会影响 array 吗？会，共享底层数组。
3. 为什么 `sum[:]` 合法？`sum` 是可寻址数组变量，可建立 slice 视图。
4. 若函数返回 `sum[:]` 是否安全？Go 会确保底层数组生命周期；但调用方能修改 slice，API 所有权仍需明确。
