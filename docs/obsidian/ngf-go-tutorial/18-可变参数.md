# 18 可变参数

> [!abstract] 核心结论
> `func f(values ...T)` 让调用者传零到多个 `T`；函数体内的 `values` 是 `[]T`。已有切片要用 `f(slice...)` 展开。可变参数适合“同类、可选、顺序有意义”的尾部输入，不适合掩盖一组有名字、有不变量的配置。

## 学习目标与前置

前置：[[10-数组与切片的类型差异]]、[[11-append容量与预分配]]、[[17-多返回值与error-last]]。完成后应能：

- 声明、调用和转发可变参数；
- 解释函数体为什么收到切片，以及传参可能涉及的底层数组共享；
- 识别零参数、nil 切片、空切片三种调用的共同点与观察差异；
- 读懂 NGF 如何把动态构建的 AWS options 以 `opts...` 交给 SDK。

## 1. 语法与心智模型

可变参数只能是最后一个形参：

```go
func sum(prefix string, values ...int) int
```

以下调用都合法：

```go
sum("a")
sum("a", 1, 2)
nums := []int{1, 2}
sum("a", nums...)
```

`nums` 不加 `...` 时是“一个 `[]int`”，不是若干个 `int`，类型不匹配。函数体内可 `len(values)`、range、索引，和普通切片相同。

## 2. 可独立运行的最小 demo

**可运行示例（Go 1.26.0，标准库）：**

```go
package main

import "fmt"

func join(prefix string, parts ...string) string {
	result := prefix
	for _, part := range parts {
		result += "/" + part
	}
	return result
}

func forward(parts []string) string {
	return join("api", parts...)
}

func main() {
	fmt.Println(join("api"))
	fmt.Println(join("api", "v1", "gateways"))
	fmt.Println(forward([]string{"v2", "routes"}))
}
```

运行 `go run main.go`，预期三行分别为 `api`、`api/v1/gateways`、`api/v2/routes`。已在 `go1.26.0` 下执行验证。

## 3. 常用模式

### 模式一：零到多个同类元素

日志键值、拼接、`errors.Join(errs...)` 都属于集合式输入。元素含义相同，数量自然变化。

### 模式二：转发已有切片

包装函数收到 `[]T` 后调用下游 `f(values...)`。这是展开调用，不会变成嵌套切片。注意下游若修改元素，可能影响共享的底层数组；需要隔离时先复制。

### 模式三：Functional Options

`New(opts ...Option)` 让零参数表示默认配置，多个 Option 按顺序应用。语法在本章，完整的默认值、校验和覆盖语义见 [[27-Functional-Options]]。

### 模式四：聚合错误

```go
return errors.Join(errs...)
```

空切片展开后相当于零参数，`errors.Join()` 返回 nil。这比手写“最后一个错误”保留更多失败信息，详见 [[38-errors-Join多错误聚合]]。

## 4. NGF 实例：动态 options 再展开

`internal/framework/waf/fetch/s3/s3.go:(*Fetcher).buildClient` 声明：

```go
var opts []func(*awsconfig.LoadOptions) error
```

它根据凭据、TLS transport 等条件逐步 `append` option，最后调用：

```go
cfg, err := awsconfig.LoadDefaultConfig(ctx, opts...)
```

**这两行是生产源码的缩略节选，不是独立 demo。** 数据路径是：

1. 凭据存在时追加静态 credentials provider，否则追加匿名 provider；
2. 自定义 transport 非 nil 时追加 HTTP client；
3. 总是追加固定 region；
4. 用 `opts...` 将切片展开给 AWS SDK 的 variadic API。

这里可变参数解决的是“选项数量运行时才知道”。如果直接写成连续嵌套调用，条件分支会重复构造逻辑。`internal/framework/waf/fetch/s3/s3_test.go` 覆盖凭据与 TLS 组合；SDK 自身负责解释每个 option。

同一仓库还有 `internal/framework/controller/register.go:Register(...Option)`：调用方直接传零个或多个选项，函数体内按顺序 range。它是下一章 [[27-Functional-Options]] 的主案例。

## 5. nil、空切片与别名边界

在函数体中，`f()` 通常得到 nil slice，而 `f([]T{}...)` 得到非 nil 空 slice；二者 `len` 都是 0，range 行为相同。不要让业务语义依赖二者差别，除非 API 明确需要保留“未提供/明确为空”。

> [!warning] 可变参数不是重载
> Go 不会因为参数个数选择另一个同名函数；只有一个函数签名，`...T` 仍要求每个元素都是 T。

- `...any` 灵活但丢掉静态约束，尤其容易出现日志键值奇数个；
- 不能把可变参数放在中间；后面若还有独立概念，应改为配置 struct；
- option 顺序若影响结果，必须在文档和测试中固定；
- callee 若 append 到收到的 slice，是否覆盖调用方底层数组取决于容量，不应把这种偶然性作为 API 契约。

## 6. 练习与验证

1. 给 demo 添加 `func safe(parts ...string)`，进入函数先 `clone := append([]string(nil), parts...)`，再修改 `clone[0]`。检查调用方切片保持不变。
2. 尝试 `join("api", []string{"v1"})`。检查：编译失败；加 `...` 后成功。
3. 将 `join` 改成接收 `[]string`。比较：调用动态切片更直接，但调用少量字面量变啰嗦；这就是两种 API 的选择依据。
