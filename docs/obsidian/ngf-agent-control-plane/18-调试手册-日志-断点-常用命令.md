# 调试手册 日志 断点 常用命令

本篇把常见现象映射到命令、日志和源码位置。

## 一键确认环境

```bash
kubectl config current-context
kubectl get pods,svc,gatewayclass,gateway,httproute -A -o wide
kubectl get cm gateway-nginx-agent-config -n default -o yaml
kubectl get deploy gateway-nginx -n default -o yaml
kubectl logs -n nginx-gateway deploy/ngf-nginx-gateway-fabric --tail=120
```

## 关键日志到源码

| 日志关键词 | 说明 | 主要源码 |
|---|---|---|
| `Starting manager` | NGF manager 启动 | `internal/controller/manager.go` |
| `Creating/Updating nginx resources` | Provisioner 创建数据面对象 | `internal/controller/provisioner` |
| `Creating connection for nginx pod` | Agent 调用 CreateConnection | `internal/controller/nginx/agent/command.go` |
| `Successfully connected to nginx agent` | Subscribe 找到 Deployment 并建立长流 | `commandService.Subscribe` |
| `Sending initial configuration to agent` | 新订阅者拿初始配置 | `setInitialConfig` |
| `NGINX configuration was successfully updated` | eventHandler 完成配置更新 | `internal/controller/handler.go` |
| `TLS files updated` | 连接因证书更新被重置 | gRPC server / `resetConnChan` |

## 场景：Agent 连不上 NGF

检查：

```bash
kubectl get svc -n nginx-gateway ngf-nginx-gateway-fabric
kubectl get cm gateway-nginx-agent-config -n default -o yaml
kubectl get secret gateway-nginx-agent-tls -n default
kubectl logs -n nginx-gateway deploy/ngf-nginx-gateway-fabric | rg 'connection|token|TLS|Unauthenticated|error'
```

重点：

- host 是否为 `ngf-nginx-gateway-fabric.nginx-gateway.svc`。
- server_name 是否一致。
- token audience 是否一致。
- TLS secret 是否存在。

## 场景：连接成功但没配置

检查：

```bash
kubectl logs -n nginx-gateway deploy/ngf-nginx-gateway-fabric | rg 'Sending initial configuration|Successfully configured|configuration'
kubectl exec -n default gateway-nginx-5f95f75958-tn9fw -- nginx -T
kubectl describe gateway gateway -n default
kubectl describe httproute coffee -n default
```

重点：

- Subscribe 是否成功。
- NGF 是否生成了配置。
- Agent 是否 ACK。
- Gateway/HTTPRoute 是否 Accepted/Programmed。

## 场景：路由不生效

检查：

```bash
kubectl get httproute -n default -o yaml
kubectl get endpointslice -n default
kubectl exec -n default gateway-nginx-5f95f75958-tn9fw -- nginx -T | rg 'cafe.example.com|coffee|tea|location|upstream'
curl -H 'Host: cafe.example.com' http://127.0.0.1:31437/coffee
curl -H 'Host: cafe.example.com' http://127.0.0.1:31437/tea
```

重点：

- Host header 是否正确。
- HTTPRoute path match 是否正确。
- backend Service 是否有 endpoints。
- NGINX 配置是否包含对应 upstream。

## 场景：改代码后协议不匹配

检查：

```bash
cd agent
make generate
make unit-test

cd ../nginx-gateway-fabric
make unit-test
```

重点：

- proto 生成代码是否同步。
- NGF 引用的 pb package 是否和 Agent 一致。
- 新字段是否有默认行为。
- stream 消息类型 switch 是否覆盖新类型。

## 使用 CodeGraph 快速定位

两个仓库都有 `.codegraph/`，优先用：

```bash
cd nginx-gateway-fabric
codegraph explore "StartManager CommandService Subscribe DeploymentStore Broadcaster"

cd ../agent
codegraph explore "CommandPlugin Subscribe receiveCallback ResourceID ConfigApply"
```
