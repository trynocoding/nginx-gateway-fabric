# NGF 与 Agent 控制面交互学习路线

这套笔记的目标是把 NGINX Gateway Fabric 与 NGINX Agent 的交互流程吃透：从当前 kind 环境中的真实 Pod、Service、ConfigMap 和日志出发，对照 NGF 与 Agent 两个 Go 项目的源码，最后落到可二次开发的改动入口。

> [!tip] 阅读方法
> 先沿着主链路读 `01` 到 `10`。如果目标是改代码，再读 `15` 到 `18`。如果目标是复盘设计原则，最后读 `19`。

## 主线地图

1. [[01-当前实验环境与资源拓扑]]：确认当前实验现场，避免源码分析脱离运行事实。
2. [[02-NGF与Agent整体架构图]]：建立控制面、数据面、gRPC、Kubernetes API 的总图。
3. [[03-NGF控制面启动流程]]：从 `StartManager` 看 NGF 如何装配控制器、事件循环、Provisioner 和 gRPC server。
4. [[04-数据面Pod是如何被Provisioner创建的]]：理解数据面 Deployment、agent 配置、TLS、token 和挂载目录。
5. [[05-Agent启动与插件总线机制]]：理解 Agent 如何加载配置、启动 MessagePipe、注册插件。
6. [[06-gRPC-MPI协议与pb生成代码]]：理解 Management Plane Interface 的 proto 和生成代码边界。
7. [[07-连接建立-CreateConnection全链路]]：理解 Agent 首次连接 NGF 的身份注册过程。
8. [[08-订阅长流-Subscribe与配置下发]]：理解双向流、广播器、初始配置和后续更新。
9. [[09-文件拉取-FileService与配置文件交付]]：理解为什么配置下发拆成文件摘要和按需拉取。
10. [[10-配置应用-ACK-状态回传]]：理解 Agent 如何应用配置、回 ACK，NGF 如何更新状态。
11. [[11-GatewayAPI到NGINX配置生成链路]]：理解 Gateway API 资源如何变成 NGINX 配置。
12. [[12-Cafe示例端到端溯源]]：用当前 Cafe demo 串起完整链路。
13. [[13-TLS-Token-鉴权与连接重置]]：理解连接安全、service account token、TLS 热更新。
14. [[14-ResourceID与数据面身份识别]]：理解 Agent 如何生成并上报数据面身份。
15. [[15-二次开发指南-改协议]]：改 proto、生成代码、实现 server/client 的步骤。
16. [[16-二次开发指南-改NGF控制面]]：改控制器、图构建、配置生成、Provisioner、状态更新。
17. [[17-二次开发指南-改Agent插件]]：改插件、消息总线和配置应用逻辑。
18. [[18-调试手册-日志-断点-常用命令]]：把日志、kubectl、断点和源码位置对应起来。
19. [[19-设计原则总结]]：提炼 NGF 与 Agent 交互背后的工程原则。
20. [[20-PodIP字段索引与controller-runtime缓存机制]]：从第一性原理理解 FieldIndexer，并核对 PodIP 校验迁移到 UUID 后留下的历史索引。
21. [[21-Processor与EventHandler调用链分析]]：追踪事件处理器从注册、分组、派发到业务回调的运行链路。
22. [[22-DeploymentBroadcaster广播器机制与全链路]]：深挖 `broadcast.go` 的 fan-out、完成屏障、并发锁与配置下发闭环。
23. [[99-源码索引与术语表]]：查源码入口和核心术语。

## 本套笔记的写法

每篇都尽量保持同一结构：

- 当前环境事实：用正在运行的 `kind-ngf-demo` 对照。
- 源码入口：明确从哪个文件、哪个函数开始读。
- 流程图：用 Mermaid 固化调用关系。
- 设计解释：解释为什么这样做，而不只是代码做了什么。
- 二开提示：指出改动入口、生成命令和风险点。
